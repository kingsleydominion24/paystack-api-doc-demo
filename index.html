3. How It Works (Simplified)

Here’s a non-technical explanation for anyone:

  1. You click the “Pay Now” button.
  2. Paystack opens a secure checkout popup.
  3. You enter card info (sandbox/test card).
  4. Paystack processes the payment safely and returns a reference number.
  5. The website shows a confirmation.

4. Frontend Code Example

For coders, here’s how it works under the hood:

<script src="https://js.paystack.co/v1/inline.js"></script>
<button id="payBtn">Pay Now</button>

<script>
document.getElementById('payBtn').addEventListener('click', function(){
  let handler = PaystackPop.setup({
    key: 'YOUR_TEST_PUBLIC_KEY', // Sandbox public key
    email: document.getElementById('email').value,
    amount: parseInt(document.getElementById('amount').value),
    currency: 'NGN',
    callback: function(response){
      alert('Payment complete! Reference: ' + response.reference);
    },
    onClose: function(){
      alert('Payment closed.');
    }
  });
  handler.openIframe();
});
</script>

5. Backend Verification (Recommended)

For real payments, your backend server verifies transactions with your secret key. Example in Node.js:

const fetch = require('node-fetch');

app.get('/verify/:reference', async (req, res) => {
  const ref = req.params.reference;
  const response = await fetch(`https://api.paystack.co/transaction/verify/${ref}`, {
    headers: { Authorization: 'Bearer YOUR_SECRET_KEY' }
  });
  const data = await response.json();
  res.json(data);
});

Backend verification ensures your site is secure and handles real money safely.

6. Tips for Beginners

  • Use sandbox/test mode first.
  • Never put secret keys in the frontend.
  • Always verify transactions on the backend.
  • Keep your UI clean and professional.
  • Document your code so others can learn from it.
