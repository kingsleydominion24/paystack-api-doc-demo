Professional Paystack Integration Guide (Test & Sandbox)

1. Overview

Paystack is a powerful payment gateway in Nigeria and Africa. This guide will teach you how to integrate Paystack into your web applications safely using sandbox/test mode so you can learn and showcase your skills.

2. Setup a Paystack Sandbox Account

  1. Go to Paystack and sign up for a developer/test account.
  2. Get your Test Public Key and Secret Key from the dashboard.
  3. Never expose the secret key on the frontend for security reasons.

3. Frontend Integration (Safe Example)

This example uses the paystack.inline.js script to initialize a payment in test mode.

<!-- Include Paystack inline script -->
<script src="https://js.paystack.co/v1/inline.js"></script>

<button id="payBtn">Pay Now</button>

<script>
document.getElementById('payBtn').addEventListener('click', function() {
    let handler = PaystackPop.setup({
        key: 'YOUR_TEST_PUBLIC_KEY', // Replace with your test public key
        email: 'customer@example.com',
        amount: 20000, // Amount in kobo (20,000 kobo = 200 NGN)
        currency: 'NGN',
        callback: function(response){
            alert('Payment complete! Reference: ' + response.reference);
        },
        onClose: function(){
            alert('Payment window closed.');
        }
    });
    handler.openIframe();
});
</script>

4. Backend Integration (Recommended)

For real payments, your backend should handle the secret key to verify transactions:

# Example in Node.js using Express
const express = require('express');
const fetch = require('node-fetch');
const app = express();

app.get('/verify/:reference', async (req, res) => {
    const reference = req.params.reference;
    const response = await fetch(`https://api.paystack.co/transaction/verify/${reference}`, {
        headers: {
            Authorization: 'Bearer YOUR_SECRET_KEY'
        }
    });
    const data = await response.json();
    res.json(data);
});

app.listen(3000, () => console.log('Server running on port 3000'));

This ensures your secret key is never exposed on the front-end.

5. Best Practices

  • Always use sandbox/test mode while learning.
  • Never put your secret key on the front-end.
  • Verify all transactions on your backend.
  • Keep your Paystack keys secure and rotate them if needed.
  • Provide a clear, professional UI for the user checkout.

6. Learn & Showcase

By following this guide, you can:

  • Create a safe, functional Paystack integration.
  • Show your GitHub Pages site with real tutorials and code snippets.
  • Use it as a portfolio project on LinkedIn to demonstrate skills.
