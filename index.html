<script>
document.getElementById('payBtn').addEventListener('click', function(){
  let email = document.getElementById('email').value;
  let amount = parseInt(document.getElementById('amount').value);
  let handler = PaystackPop.setup({
    key: 'YOUR_TEST_PUBLIC_KEY',
    email: email,
    amount: amount,
    currency: 'NGN',
    callback: function(response){
      alert('Payment complete! Reference: ' + response.reference);
    },
    onClose: function(){
      alert('Payment window closed.');
    }
  });
  handler.openIframe();
});
</script>

5. Backend Verification

Real payments should be verified in the backend for security:

# Node.js Example
const fetch = require('node-fetch');
app.get('/verify/:reference', async (req, res) => {
  const ref = req.params.reference;
  const response = await fetch(`https://api.paystack.co/transaction/verify/${ref}`, {
    headers: { Authorization: 'Bearer YOUR_SECRET_KEY' }
  });
  const data = await response.json();
  res.json(data);
});

This ensures safe handling of real money.

6. Tips & Best Practices

  • Always start in sandbox/test mode.
  • Never expose secret keys on the frontend.
  • Verify transactions on backend.
  • Keep your UI professional and simple.
  • Document your project for learning and sharing.
